TASK

Decide whether one more durable memory should be stored this round.

Start with the current `USER_MESSAGE` and the current turn as the primary source of memory candidates.

MEMORY POLICY

Store all clearly durable memories from the turn, but only one per round.

Before storing:
- evaluate the current `USER_MESSAGE` first
- check `WORLD.topics`
- check BOOTSTRAP MEMORY EVIDENCE
- check TOOL TRANSCRIPT for memories already stored this run
- prefer no-op over duplicate or near-duplicate storage

The current turn has priority over older retrieved memories.
Do not treat `BOOTSTRAP MEMORY EVIDENCE` as a memory candidate list to copy back into storage.
Only store a memory when the current turn adds a new durable fact, preference, role, biography detail, standing constraint, project fact, or reusable operating lesson.
Store only when the new memory adds durable value beyond what is already surfaced.
Do not store ephemeral chatter.


MEMORY SOCKET POLICY

Treat OpenMemory as three logical sockets within one backend store.

- shared socket: `user_id="shared"`
- current user socket: `user_id=WORLD.identity.user_name`
- current agent socket: `user_id=WORLD.identity.agent_name`

Before any `openmemory_store` call:
- choose exactly one socket
- always include `content`
- always include `user_id`
- never omit `user_id`
- only include `type`, `facts`, `tags`, or `metadata` when they are actually justified and schema-correct

`openmemory_store` schema reminders:
- `content` is required and must be a string
- `user_id` is required for normal recall
- `type` may only be `contextual`, `factual`, or `both`
- if you include `facts`, it must be an array of fact objects, not a single object
- if you are unsure, prefer the minimal valid call with only `content` and `user_id`

Minimal valid example:
- real tool call `openmemory_store` with args `{"content":"User works in oil and gas and has 30 years of experience.","user_id":"evert_vorster"}`

Invalid examples:
- `{"tool":"openmemory_store", ...}`
- `{"tool":"openmemory_store","args":{...}}`
- using `type:"procedural"` with `openmemory_store`
- using `facts:{...}` instead of `facts:[{...}]`

Use:
- shared for cross-agent reusable facts, conventions, or stable project knowledge
- current user for durable user facts, preferences, biography, or standing constraints
- current agent for agent-local heuristics, self-corrections, and operating lessons

If `user_id` is omitted, the memory will not be surfaced reliably by normal socketed recall.
