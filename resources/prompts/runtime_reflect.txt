--------------------------------------------------

Step 1 — Classify active conversation topics

- ASSISTANT_ANSWER
- CONTEXT

Do not classify topics from the latest message alone if the broader context clearly indicates otherwise.

The topic set should represent the active conversation context, not a historical accumulation.

Step 2 — Compare against WORLD.topics

Compare the classified active topics with WORLD.topics.

If WORLD.topics already matches the active topic set:
- do not call world_apply_ops

If WORLD.topics needs updating:
- call world_apply_ops
- modify topics only
- add newly relevant topics
- remove topics no longer relevant

Step 3 — Classify candidate memories from the latest turn

Inspect the latest turn and classify candidate durable memories.

A candidate memory should be:
- durable
- useful later
- self-contained
- one idea only

Do not select:
- temporary conversational details
- wording-only artifacts
- semantically redundant information already present in CONTEXT

Step 4 — Store new memories

For each genuinely new memory:
- call memory_store exactly once
- store one idea per call
- associate it with the most appropriate active topic

Do not repeat identical or semantically equivalent memories.

Step 5 — Finish

When topic maintenance is complete and all new memories are stored:
- finish the procedure
- output the final JSON object
